第1章 Vue 核心

1.Vue 简介

1.1 Vue 介绍与描述

  1. vue 是一套用于动态构建用户界面的渐进式 JavaScript 框架。
  2. 渐进式，Vue 可以自底向上逐层的应用。对于简单应用，只需要一个轻量小巧的核心库；对于复杂应用，可以引入各种各样的 Vue 插件。

1.2 Vue 的特点

1.2.1 组件化

  1. 采用组件化模式，提高代码的复用率，让代码易于维护。
  2. 组件化：将页面拆分成不同的部分，每个部分为一个组件，每个组件为一个单独的 vue文件，每个 vue 文件的标签、样式、交互进行分开管理。
  3. 提高代码的复用率：当其他的页面也需要使用相同的组件时，直接引入写好的对应的组件即可。
  4. 代码易于维护：每个vue文件不同组件的标签、样式、交互进行单独管理，不会互相影响。
  5. 例如：下面的活动、新闻和热门三个板块，都可以单独作为一个组件，给其他页面引用，提高代码复用。

1.2.2 声明式编码

  1. 声明式编码：只需声明数据，然后将数据写入对应的标签位置即可，无需操作 DOM，提高开发效率。
  2. 对应的还有命令式编码：即一条命令实现对应的步骤，一个命令一个步骤，需操作 DOM。

4.初识 Vue

  1. 想让 Vue 工作，就必须创建一个 Vue 实例，且要通过属性 el 传入一个配置对象（容器），Vue 实例与配置对象一一对应，不能相互一对多；真实开发只有一个Vue实例，并且配合着组件一起使用。

  2. root 容器里面的代码被称为【Vue模板】，这些代码依然符合 html 规范，只不过混入了一些的 Vue 写法。

  3. 在模板代码中使用 Vue 插值语法 {{}} 来包裹 JS 表达式，以此来读取到 data 中的属性，一旦 data 中的属性值发生改变，那么模版中用到该属性值的地方也会自动更新。

  4. 区分 JS 表达式和 JS 代码（语句）。

  • 表达式，一个表达式会生成一个值，可以放在任何一个需要值的地方。如下所示：

    • 变量：a
    • 变量运算：a+b
    • 调用函数：demo(1)
    • 等式：x === y ? 'a' : 'b'

  • JS 代码（语句）

    • if(){}
    • for(){}

  • JS 表达式是一种特殊的 JS 代码（语句）。

  1. 指定属性 el 的值后，就可以将指定容器中的模板交给 Vue 实例解析，Vue 实例解析好后将内容挂载到指定容器。

4.1 简单示例

<body>
    <!-- 准备一个容器 -->
    <div id="root">
        <h1>Hello，{{name}}</h1> <!--读取到 data 中的 name 属性-->
    </div>
    <script type="application/javascript">
        Vue.config.productionTip = false; // 阻止 Vue 在启动时生产生产提示
        // 创建Vue实例，并给参数配置对象赋值
        const x = new Vue({
        el: '#root', // 指定当前 Vue 容器为哪个容器服务，值通常为css选择器字符串
        // el: document.getElementById('root'), // 也可以这种方式，但是一般u不用
        data: { // data 中用于存储数据，数据供 el 指定的容器使用，值暂时写成一个对象
            name: 'World'
        }
        });
    </script>
</body>

5.模板语法

  1. Vue 模板语法主要分为两类：
  • 插值语法，即前面使用到的 {{}}
  • 指令语法，以 v- 开头，例如：v-bind，v-on 等等。

5.1 插值语法

  1. 功能：作用于标签体内容。

  2. 写法：{{xxx}}，xxx 是 JS 表达式，且可以直接读取到 data 中的数据。

  3. 代码示例

<body>
    <div id="root">
        <h1>插值语法</h1>
        <h3>你好，{{name}}</h3>
    </div>
    <script type="application/javascript">
        const vm = new Vue({
            el: '#root',
            data: {
                name: 'Jack'
            }
        });
    </script>
</body>

5.2 指令语法

  1. 功能：作用于标签，包括：标签属性，标签体内容，绑定事件等等，其形式都为：v-???；

  2. 举例：以指令 v-bind 为例，作用于标签属性，可简写为一个冒号，它将其值与一个 JS 表达式绑定起来，JS 表达式可以读取 data 中的所有属性；

  3. 代码示例：

<body>
    <div id="root">
        <h1>指令语法</h1>
        <a v-bind:href="school.url">{{school.name}}</a>
        <br/>
        <!-- 简写, 并调用 String 的函数 toUpperCase()-->
        <a :href="school.url.toUpperCase()">{{school.name}}</a>
    </div>
    <script type="application/javascript">
        const vm = new Vue({
            el: '#root',
            data: {
                school: {
                    name: '百度',
                    url: 'https://www.baidu.com',
                }
            }
        });
    </script>
</body>

9.1 Object 对象的两个方法

  1. 数据代理的底层实现涉及到Object 对象的两个方法：Object.defineProperty() 和 Object.defineProperties()方法。它们用于给对象添加属性。

  2. Object.defineProperty() 用于单个给对象添加属性，它需要传入三个参数：

  • 需要进行属性添加的对象。
  • 将要添加到对象的新属性名。
  • 配置对象，在配置对象中分别可以通过 value 属性指定新属性的值；通过 enumerable 属性控制属性是否可以枚举（参与遍历），默认 false；通过 writable 属性控制属性是否可以被修改，默认 false；通过 configurable 属性控制属性是否可以被删除，默认 false。

代码示例：

let person = {
    name: '张三',
    sex: '男'
}

Object.defineProperty(person, 'age', { // 添加 age 属性
    value: 18,
    enumerable: false, // 控制属性是否可以枚举，默认 false
    writable: true, // 控制属性是否可以被修改，默认 false
    configurable: true // 控制属性是否可以被删除，默认 false
})

  • get() 函数，在配置对象中配置 get() 函数后，当读取配置的属性值时，get() 函数会被调用，去重新获取该属性的值。因此我们可以将配置的属性的值与其他变量的值绑定起来。如下，将配置的属性 age 的值与 属性 num 的值绑定：

10.事件处理

10.1 事件绑定及数据传递

  1. Vue 提供了 v-on: 事件绑定指令，用来辅助程序员为 DOM 元素绑定事件监听，v-on: 简写为 @。原生的 DOM 对象有 onclick、oninput 等原生事件，在 Vue 中可以使用 v-on:click、v-on:input 等来监听 DOM 元素的对应事件，并为其绑定相应的事件处理函数，其中事件处理函数需要在 Vue 实例对象的methods 节点中进行声明。
<body>
<div id="root">
    <button v-on:click="showInfo">点我提示信息</button>
</div>
</body>
<script type="application/javascript">
    const vm = new Vue({
        el: '#root',
        data: {},
        methods: {
            showInfo() {
                console.log(this) // 此处的this是vm
            }
        }
    });
</script>
  1. 事件处理函数不进行参数的传递，默认会有一个参数，即事件的触发对象。
<body>
    <div id="root">
        <button v-on:click="showInfo">点我提示信息</button>
    </div>
</body>
<script type="application/javascript">
    const vm = new Vue({
        el: '#root',
        data: {},
        methods: {
            showInfo(event) {
                // 获取事件源的标签体内容
                console.log(event.target.innerHTML);
            }
        }
    });
</script>
  1. 传递自定义参数，注意此时事件的触发对象参数需要显式传递，参数位置不固定，否则调用时报错。
<body>
    <div id="root">
        <button @click="showInfo($event, 66)">点我提示信息</button>
    </div>
</body>
<script type="application/javascript">
    const vm = new Vue({
        el: '#root',
        data: {},
        methods: {
            showInfo(event, param) {
                console.log(event.target.innerHTML)
                console.log(param)
            }
        }
    });
</script>
  1. method的this指向
  • 事件处理函数如果写成普通函数，则事件处理函数的 this 指向 vue 实例对象；
  • 如果事件处理函数写成箭头，则事件处理函数的 this 指向 windows 对象，因为箭头函数没有自己的 this。
<body>
    <div id="root">
        <button @click="showInfo($event, 66)">点我提示信息</button>
    </div>
</body>
<script type="application/javascript">
    const vm = new Vue({
        el: '#root',
        data: {},
        methods: {
            showInfo: (event, param)=> {
                console.log(param)
                console.log(event.target.innerHTML)
                console.log(this)   // Window {...}
            }
        }
    });

10.2.3 事件的捕获与冒泡

  1. 事件的捕获是自上向下或由外而内进行的，即从 window 开始，向触发事件的元素传递事件。

11.4 计算属性实现

  1. 属性，在 Vue 中只要是写在 data 对象中的配置项，都认为是属性。

  2. 计算属性，即将调用原有的 data 中的属性，生成的新的属性即为计算属性。在 Vue 中，计算属性定义在配置对象 computed 中，它的值通过实现一个 get() 函数获取。

  3. 计算属性的值在计算完成后，最终会被添加到 Vue 的实例上。使用计算属性的方式与使用 data 中的属性的方式一样，直接使用即可，不需要通过其 get() 调用。

  4. get() 函数被调用的时机。

    • 初次调用计算属性时，get() 函数会被调用来获取计算属性的值，并将值缓存起来。

    • get() 方法中依赖的属性的值发生了变化，此后再次读取对应的计算属性的值时，get() 会被调用，以重新获取计算属性的值。

  5. set() 函数，修改计算属性时需要在计算属性中定义set()函数，当计算属性被修改时，set() 函数会被调用，并且 set() 函数中需要修改的是计算属性所依赖的属性的值。

  6. 代码示例：

<body>
    <div id="root">
        姓：<input type="text" v-model="firstName"><br/>
        名：<input type="text" v-model="lastName"><br/>
        全名：<span>{{fullName}}</span>
    </div>
</body>
<script type="application/javascript">
    let vm = new Vue({
        el: '#root',
        data: {
            firstName: '张',
            lastName: '三',
        },
        computed: {
            fullName: {
                get() { // 计算属性的 get() 函数中的 this 就是 Vue 实例 vm，所以直接用 this 调用属性
                    return this.firstName.slice(0, 3).split('').reverse().join('') + '-' + this.lastName;
                },
                set(value) {
                    const arr = value.split('-');
                    this.firstName = arr[0];
                    this.lastName = arr[1];
                }
            }
        }
    })
</script>

计算属性底层借助了 Object.defineproperty 方法提供的getter和setter

  1. 计算属性相比于 methods 的实现方式更好，因为内部有缓存机制，若 get() 方法中依赖的属性的值没有发生变化，调用计算属性时会直接获取缓存中的值，效率更高，调试方便。

  2. 计算属性的简写。计算属性更多是用于数据的展示，很少用于数据的修改。当我们确定计算属性只读不改，此时我们可以使用计算属性的简写形式，即把计算属性写成函数形式，此时计算属性函数相当于之前对象形式写法里面的get()函数。代码示例如下：

15.3 面试题：react、vue中的key有什么作用？（key的内部原理）

虚拟DOM中key的作用：

  • key是虚拟DOM对象的标识，当数据发生变化时，Vue会根据【新数据】生成【新的虚拟DOM】,
  • 随后Vue进行【新虚拟DOM】与【旧虚拟DOM】的差异比较，比较规则如下：

对比规则：

  • 旧虚拟DOM中找到了与新虚拟DOM相同的key：
    • 若虚拟DOM中内容没变, 直接使用之前的真实DOM！
    • 若虚拟DOM中内容变了, 则生成新的真实DOM，随后替换掉页面中之前的真实DOM。
  • 旧虚拟DOM中未找到与新虚拟DOM相同的key
    • 创建新的真实DOM，随后渲染到到页面。

用index作为key可能会引发的问题：

  • 若对数据进行：逆序添加、逆序删除等破坏顺序操作:
    • 会产生没有必要的真实DOM更新 ==> 界面效果没问题, 但效率低。
  • 如果结构中还包含输入类的DOM：
    • 会产生错误DOM更新 ==> 界面有问题。

开发中如何选择key?:

  • 最好使用每条数据的唯一标识作为key, 比如id、手机号、身份证号、学号等唯一值。
  • 如果不存在对数据的逆序添加、逆序删除等破坏顺序操作，仅用于渲染列表用于展示，
    • 使用index作为key是没有问题的。

school组件本质是一个名为VueComponent的构造函数，且不是程序员定义的，是Vue.extend生成的。

我们只需要写<school/>或<school></school>，Vue解析时会帮我们创建school组件的实例对象，

  • 即Vue帮我们执行的：new VueComponent(options)。

特别注意：每次调用Vue.extend，返回的都是一个全新的VueComponent！！！！

关于this指向：

  • 组件配置中：
    • data函数、methods中的函数、watch中的函数、computed中的函数 它们的this均是【VueComponent实例对象】。
  • new Vue(options)配置中：
    • data函数、methods中的函数、watch中的函数、computed中的函数 它们的this均是【Vue实例对象】。

VueComponent的实例对象，以后简称vc（也可称之为：组件实例对象）。

  • Vue的实例对象，以后简称vm。
