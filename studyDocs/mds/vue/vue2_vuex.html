vuex

  • vue2对应vuex版本vue@^3
  • npm i vuex@^3 --- 安装vuex的3的最新版

在Vue中实现集中式状态（数据）管理的一个Vue插件，对vue应用中多个组件的共享状态进行集中式的管理（读/写），也是一种组件间通信的方式，且适用于任意组件间通信。

使用场景：多个组件需要共享数据时

1.基本使用

1.1 创建入口文件

js或者ts文件，我这里为了统一和文件结构简洁：src/store/index.js

// 该文件用于创建Vuex中最核心的store
import Vuex from 'vuex'
import Vue from 'vue'
//应用Vuex插件
Vue.use(Vuex)
// 准备 actions --- 用于响应组件中的动作
const actions = {
    // actions 能拿到 context，可以直接操作数据。但不要这么做
    // 为什么不在 actions 这一步就操作数据？因为 vue development tool 只监视 mutations，也就是在 actions 直接操作，我们就调试不了了
    actions1 (context, value) {
      // 提交给其他actions, 也是业务代码里面实际修改store的命令
      context.dispatch('actions函数名', value)
      // 提交给mutations，执行更改store
      context.commit('mutation函数名', value)
    },
}
// 准备 mutations --- 用于操作数据（状态）
const mutations = {
    mutations1:function(state, value) {
        //操作state
        state.data1 = value;
    },
    //...其他函数
}
// 准备 state --- 用于存储数据(状态)
const state = {
    data1: 2
    // 数据1:xxx,
    //...其他数据
}

// 创建 store
export default new Vuex.Store({
  actions,
  mutations,
  state,
})

1.2 注册使用vuex

vue入口文件(一般是src/main.js)进行插件、vuex、vue-router的注册和使用

// main.js
import Vue from "vue";
import App from "./App.vue";

import store from "./store";   // 这里是简写了index.js

Vue.config.productionTip = false;

// 创建vm
new Vue({
  el: "#app",
  render: (h) => h(App),
  store
});

1.3 组件读取 vuex 中的数据

  • this.$store.state.data1
  • {{$store.state.data1}}

1.4 组件中修改 vuex 中的数据

  • 推荐：this.$store.dispatch({action函数名}, 1)
    • vc先用dispatch 先跟 actions 对话，然后 actions 再跟 mutations 对话，mutations去修改state属性值
  • 不推荐：this.$store.commit({mutation函数名}, 1)
    • vc用commit 直接跟 mutations 对话，相当于跳过了 actions
  • 超级不推荐：this.$store.state.data1 = 1
    • 直接操作 store。其他人没法复用这个逻辑

2.getters

当state中的数据需要经过加工后再使用时，可以使用getters加工

2.1 添加getters

// src/store/index.js
// 创建 getters
const getters = {
  bigSum(state) {
    return state.sum * 10;
  },
};
// 创建并暴露 store 时配置 getters
export default new Vuex.Store({
  actions,
  mutations,
  state,
  getters,
});

2.2 读取 getters 数据

  • this.$store.getters.bigSum

3.四个mapxxx方法

当一个组件需要获取多个状态的时候,每次书写使用修改有些重复和冗余

  • 引入
import { mapState, mapGetters, mapMutations, mapActions} from 'vuex';

3.1 mapState

用于帮助我们映射state中的数据为计算属性

  computed: {
      //借助mapState生成计算属性：sum、school、subject（对象写法）
      ...mapState({sum1:'sum',school:'school',subject:'subject'}),
      // this.sum1,this.school
      //借助mapState生成计算属性：sum、school、subject（数组写法：生成的计算属性名要和state的属性名一致，下面的数组写法同理）
      ...mapState(['sum','school','subject']),
      // this.sum,this.school
      //函数写法：适用嵌套属性
      mapState({ sum: state => state.xxx.sum });
  },

3.2 mapGetters

用于帮助我们映射getters中的数据为计算属性

computed: {
    //借助mapGetters生成计算属性：bigSum（对象写法）
    ...mapGetters({bigSum:'bigSum'}),
    // this.bigSum
    //借助mapGetters生成计算属性：bigSum（数组写法）
    ...mapGetters(['bigSum'])
},

3.3 mapActions

用于帮助我们生成与actions对话的方法

methods: {
    //借助mapActions生成计算属性：incrementIfOdd、incrementWait（对象写法）
    ...mapActions({incrementIfOdd: 'incrementIfOdd',incrementWait: 'incrementWait',}),

    // this.incrementIfOdd(1)
    // <div @click="incrementIfOdd(1)"></div>

    //借助mapActions生成计算属性：incrementIfOdd、incrementWait（数组写法）
    ...mapActions(['incrementIfOdd', 'incrementWait'])
},

3.4 mapMutations

用于帮助我们生成与mutations对话的方法

methods:{
    //靠mapMutations生成：increment、decrement（对象形式）
    ...mapMutations({increment:'INCREMENT',decrement:'DECREMENT'}),
    // 不推荐，最好先经过actions
    // this.increment(1)
    // <div @click="increment(1)"></div>
    ...mapMutations(['INCREMENT','DECREMENT']),
}

3.5 特别注意

mapActions与mapMutations使用时，若需要传递参数需要：在模板中绑定事件时传递好参数，否则参数是事件对象。

4.模块化和命名空间

模块化：让代码更好维护，让多种数据分类更加明确

命名空间: 避免命名冲突、提高代码可读性和维护性、模块隔离、配合 mapxxxx更方便

4.1 设置

修改Vuex中各个文件,添加namespaced：true，之后在不同的页面中引入 getters、actions、mutations、state 时，需要加上所属的模块名。

// src/store/countAbout/index.js
const countAbout = {
   namespaced: true, //开启命名空间
   state:{
    ....
   },
   mutations: {
    ....
   },
   actions:{
   ....
   },
   getters:{
    ...
   }
}

// src/store/personAbout/index.js
const personAbout = {
   namespaced: true, //开启命名空间
   state:{
    ....
   },
   mutations:{
    ....
   },
   actions:{
    ....
   },
   getters:{
    ...
   }
}
// src/store/index.js
const store = new Vuex.Store({
   modules: {
      countAbout:countAbout,
      personAbout:personAbout,
   }
})

4.2 使用

4.2.1 读取属性

//方式一：直接读取
this.$store.state.personAbout.list
//方式二：借助mapState读取
...mapState('countAbout',['sum','school','subject'])

4.2.2 获取getters

//方式一：直接读取
this.$store.getters['personAbout/firstPersonName']
//方式二：借助mapGetters读取
...mapGetters('countAbout',['bigSum'])

4.2.3 调用action

//方式一：自己直接dispatch
this.$store.dispatch('personAbout/addPersonWang',person)
//方式二：借助mapActions
...mapActions('countAbout',{
   incrementOdd:'jiaOdd',incrementWait:'jiaWait'})

4.2.4 调用mutation

//方式一：自己直接commit
this.$store.commit('personAbout/ADD_PERSON',person)
//方式二：借助mapMutations
...mapMutations('countAbout',{
   increment:'JIA',decrement:'JIAN'})
