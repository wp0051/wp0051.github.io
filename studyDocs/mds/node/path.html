前端笔记

Path

用于处理和转换文件路径

path.resolve([...paths])

将路径片段解析为一个绝对路径。它会从右到左处理路径片段，直到构建出一个绝对路径。如果处理完所有路径片段后仍未生成绝对路径，则会使用当前工作目录 (process.cwd()) 作为基础路径

console.log(path.resolve('foo', 'bar', 'baz')); 
// 假设当前工作目录是 /home/user
// 输出: /home/user/foo/bar/baz

console.log(path.resolve('/foo', 'bar', 'baz')); 
// 输出: /foo/bar/baz (遇到绝对路径 /foo 后，后面的 bar, baz 被追加)

console.log(path.resolve('..', 'siblingDir')); 
// 输出: /home/user/siblingDir (从当前工作目录的父目录开始)

path.relative(from, to)

根据当前工作目录，计算从 from 路径到 to 路径的相对路径

const from = '/data/orandea/test/aaa';
const to = '/data/orandea/impl/bbb';
console.log(path.relative(from, to)); 
// 输出: ../../impl/bbb

path.join([...paths])

将所有给定的路径片段使用平台特定的分隔符（Windows 上是 \，POSIX 上是 /）连接在一起，然后规范化生成的路径。它会处理 .. 和 . 片段。

console.log(path.join('/foo', 'bar', 'baz/asdf', 'quux', '..')); 
// 输出: /foo/bar/baz/asdf (.. 会向上一级目录)

console.log(path.join('foo', 'bar', 'baz/asdf')); 
// 输出: foo/bar/baz/asdf (相对路径)

path.basename(path[, ext])

返回路径的最后一部分（文件名或目录名）。可以可选地提供一个扩展名，如果路径的最后一部分以该扩展名结尾，则将其从结果中移除。

console.log(path.basename('/foo/bar/baz/asdf.js')); 
// 输出: asdf.js

console.log(path.basename('/foo/bar/baz/asdf.js', '.js')); 
// 输出: asdf

path.dirname(path)

返回路径的目录名部分

console.log(path.dirname('/foo/bar/baz/asdf.js')); 
// 输出: /foo/bar/baz

path.extname(path)

返回路径中文件扩展名的部分。扩展名是从最后一次出现 . 开始到字符串结尾的部分。如果路径中没有 . 或最后一个 . 是第一个字符，则返回空字符

console.log(path.extname('index.html')); 
// 输出: .html

console.log(path.extname('index.coffee.md')); 
// 输出: .md

console.log(path.extname('index.')); 
// 输出: .

console.log(path.extname('index')); 
// 输出: (空字符串)
