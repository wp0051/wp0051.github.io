安装多个版本的node

创建一个弄得文件夹，创建多个版本的文件夹(win10好用一些)

1.先安装一个低版本的node，然后在电脑>属性>高级系统设置>path添加，在命令行里面输入node -v验证 2.输入where node查看路径(在自带的命令行，不要用powershell)之后把这个低版本所在的文件夹改名字，让注册表找不到 3.再安装高版本，按照上面的执行一边，现在node -v就是高版本的了 4.然后把低版本的文件夹名字改回来，这时候node -v就是旧版本 5.win10的path有上移下移的功能，通过上下移动调整当前版本

使用包(来源：https://fengmk2.com/blog/2014/03/node-env-and-faster-npm.html)

1.假设将git项目存放在 D:\gitItem 里面，在 git bash here里; cd gitItem; git clone https://github.com/cnpm/nvmw.git;下载成功 2.在命令提示符(cmd),设置环境变量 set "PATH=d:/gitItem/nvmw;%PATH%";使用 nvmw 检验 3.nvmw ls 查看安装的node版本,使用 nvmw use 8.11.0; 如果一直失败，报 0 Unknown 需要把 nvmw/fget.js 文件打开 ，将 Msxml2.XMLHttp替换为 Msxml2.ServerXMLHttp 4.nvmw.bat 文件22~28行用下面的代码替换

if not defined NVMW_NODEJS_ORG_MIRROR (
 set "NVMW_NODEJS_ORG_MIRROR=http://npm.taobao.org/mirrors/node"
)

if not defined NVMW_IOJS_ORG_MIRROR (
 set "NVMW_IOJS_ORG_MIRROR=http://npm.taobao.org/mirrors/iojs"
)

5.nvmw.bat 文件137行 : x64改为win-x64 6.get_npm.js 文件第8行用下面的代码替换

var NVMW_NPM_MIRROR = process.env.NVMW_NPM_MIRROR || 'http://npm.taobao.org/mirrors/npm';

7.get_npm.js 文件第44~51行用下面的代码替换

 var pkgUri = "https://npm.taobao.org/mirrors/node/index.json";
  wget(pkgUri, function (filename, pkg) {
    if (filename === null) {
      return noNpmAndExit();
    }
    var _pkg = JSON.parse(pkg);
    for(var i = 0,n=_pkg.length;i<n;i++){
        var obj = _pkg[i];
        if(obj.version == binVersion){
            downloadNpmZip(obj.npm);
        }
    }
  })

8.上面的文件修改成功之后重启你的命令行终端，NVMW的常用指令

//查看是否可以运行 nvmw
nvmw
//查看已安装的包  这里如果是第一次查看是什么都没有的主要是为了与安装后的作比较
nvmw ls
//安装任意版本的 node  安装时间会有一点的久，注意：安装之后默认的使用的就是你安装的这个版本了
nvmw install v4.8.6
//安装之后再次查看已安装的包 这时会发现有了 4.8.6 的包
nvmw  ls
//查看 node 版本，会发现已经是 v4.8.6 了 证明成功了
node -v
//卸载某个版本
nvmw uninstall v0.10.20
//使用某个版本，前提是你使用的这个版本已安装
nvmw use v0.10.20
//查看所有的你已安装的版本包
nvmw ls
遇到的坑

1.每次使用node必须先在命令行里使用 nvmw use [version]; 2.如果安装nvmw安装之前电脑里面的npm版本低的话，是不支持安装高版本的有兼容，要安装其他版本，需要在解压压缩包之后，强制结束安装，再安装已经安装成功的版本的结构，来把新版本设置好，之后就可以用了。 3.在执行nvmw use [version];之后才有npm，但是使用 npm 安装的包，只能在 安装这个包时的node版本 use 的情况下，才能使用该包

*** 就是npm和node版本绑定，包和npm的版本绑定，即 包 、 npm 、 node 是绑定的 *** *** 所以安装的包都在对应node版本文件夹的node_modules下面 ***

linux(来源：https://fengmk2.com/blog/2014/03/node-env-and-faster-npm.html)

1.假设大家都使用 ~/git 目录存放 git 项目 cd ~/git; git clone https://github.com/cnpm/nvm.git 2.在 ~/.bashrc, ~/.bash_profile, ~/.profile, 或者 ~/.zshrc 文件添加以下命令: source ~/git/nvm/nvm.sh (使用vim) 3.重启服务器使用 nvm 检验 4.使用 nvm ls-remote; 展示所有可以下载的 node版本 5.使用 nvm install 8.11.0; 可以一直使用这个命令安装多个node;
