if not defined NVMW_NODEJS_ORG_MIRROR (

nvmw.bat 文件137行 : x64改为win-x64

get_npm.js 文件第8行用下面的代码替换

var NVMW_NPM_MIRROR = process.env.NVMW_NPM_MIRROR || 'http://npm.taobao.org/mirrors/npm';

get_npm.js 文件第44~51行用下面的代码替换

 var pkgUri = "https://npm.taobao.org/mirrors/node/index.json";
  wget(pkgUri, function (filename, pkg) {
    if (filename === null) {
      return noNpmAndExit();
    }
    var _pkg = JSON.parse(pkg);
    for(var i = 0,n=_pkg.length;i<n;i++){
        var obj = _pkg[i];
        if(obj.version == binVersion){
            downloadNpmZip(obj.npm);
        }
    }
  })

上面的文件修改成功之后重启你的命令行终端，NVMW的常用指令

//查看是否可以运行 nvmw
nvmw
//查看已安装的包  这里如果是第一次查看是什么都没有的主要是为了与安装后的作比较
nvmw ls
//安装任意版本的 node  安装时间会有一点的久，注意：安装之后默认的使用的就是你安装的这个版本了
nvmw install v4.8.6
//安装之后再次查看已安装的包 这时会发现有了 4.8.6 的包
nvmw  ls
//查看 node 版本，会发现已经是 v4.8.6 了 证明成功了
node -v
//卸载某个版本
nvmw uninstall v0.10.20
//使用某个版本，前提是你使用的这个版本已安装
nvmw use v0.10.20
//查看所有的你已安装的版本包
nvmw ls

遇到的坑

1.每次使用node必须先在命令行里使用 nvmw use [version]; 2.如果安装nvmw安装之前电脑里面的npm版本低的话，是不支持安装高版本的有兼容，要安装其他版本，需要在解压压缩包之后，强制结束安装，再安装已经安装成功的版本的结构，来把新版本设置好，之后就可以用了。 3.在执行nvmw use [version];之后才有npm，但是使用 npm 安装的包，只能在 安装这个包时的node版本 use 的情况下，才能使用该包

*** 就是npm和node版本绑定，包和npm的版本绑定，即 包 、 npm 、 node 是绑定的 *** *** 所以安装的包都在对应node版本文件夹的node_modules下面 ***

linux

参考文章

假设大家都使用 ~/git 目录存放 git 项目 cd ~/git; git clone https://github.com/cnpm/nvm.git

在 ~/.bashrc, ~/.bash_profile, ~/.profile, 或者 ~/.zshrc 文件添加以下命令: source ~/git/nvm/nvm.sh (使用vim)

重启服务器使用 nvm 检验

使用 nvm ls-remote; 展示所有可以下载的 node版本

使用 nvm install 8.11.0; 可以一直使用这个命令安装多个node;

nvm --version 如果不存在 解决

  1. 重启终端
  2. 缺少全局设置
cd ~/
open .bashrc (不存在，先创建 touch .bashrc)
  1. 打开 .bashrc 输入
export NVM_DIR="$HOME/.nvm"
[ -s "$NVM_DIR/nvm.sh" ] && \. "$NVM_DIR/nvm.sh"  # This loads nvm
[ -s "$NVM_DIR/bash_completion" ] && \. "$NVM_DIR/bash_completion"  # This loads nvm bash_completion
  1. 执行 source .bashrc,重启配置

问题汇总

每次关掉终端，再重新打开的时候执行nvm又会提示 command not find。执行上面的脚本之后又可正常使用，如此反复,也可能是因为使用的终端是zsh的，问题待查

考虑应该是m1的原因 使用.bashrc无法长期有效， 需要用到 .zshrc

apple 芯片 安装低版本的node安装不上，是因为低版本不兼容apple芯片，需要 Rosetta 2转译

终端执行 arch -x86_64 zsh 终端进入 Rosetta 2转译，然后进行安装（需要.bashrc文件添加环境变量）

常规操作指令

nvm ls # 列出当前安装的所有node版本
nvm ls-remote # 列出所有node版本

nvm install 版本号 # 安装指定版本的node 版本号取nvm ls-remote列出的

nvm uninstall version  # 卸载指定版本号

nvm use version # 切换到指定node版本

设置默认node版本

通过以上方式安装nvm后，切换好node版本，但是每次打开vscode后，版本号会变掉，这个时候可以设置默认版本

nvm alias default version

npm install --unsafe-perm
